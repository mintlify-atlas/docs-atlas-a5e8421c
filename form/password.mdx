## Overview

The Password component provides a secure password input with show/hide functionality and optional strength validation for new passwords.

## Import

```jsx
import { Password } from '@jeremiemeunier/form';
```

## Basic Usage

```jsx
import { Password } from '@jeremiemeunier/form';
import { useState } from 'react';

function MyForm() {
  const [passwordContent, setPasswordContent] = useState({
    error: false,
    message: '',
    value: ''
  });

  return (
    <Password
      label="Password"
      content={passwordContent}
      setContent={setPasswordContent}
    />
  );
}
```

## Password Confirmation

```jsx
const [passwordContent, setPasswordContent] = useState({
  error: false,
  message: '',
  value: ''
});

const [confirmContent, setConfirmContent] = useState({
  error: false,
  message: '',
  value: ''
});

const validatePasswordMatch = () => {
  if (passwordContent.value !== confirmContent.value) {
    setConfirmContent(prev => ({
      ...prev,
      error: true,
      message: 'Passwords do not match'
    }));
    return false;
  }
  return true;
};

return (
  <>
    <Password
      label="New Password"
      content={passwordContent}
      setContent={setPasswordContent}
      isNew={true}
    />
    <Password
      label="Confirm Password"
      content={confirmContent}
      setContent={setConfirmContent}
      placeHolder="Re-enter your password"
    />
  </>
);
```

## Props

<ParamField path="label" type="string" required>
  Label text displayed above the password input
</ParamField>

<ParamField path="content" type="PasswordContent" required>
  State object containing `error`, `message`, and `value` (string) properties
</ParamField>

<ParamField path="setContent" type="React.Dispatch<React.SetStateAction<PasswordContent>>" required>
  State setter function for the content object
</ParamField>

<ParamField path="isNew" type="boolean" default="false">
  Shows password strength validation criteria when true
</ParamField>

<ParamField path="placeHolder" type="string">
  Placeholder text shown when input is empty
</ParamField>

<ParamField path="size" type="number">
  Width of the password input in grid units
</ParamField>

<ParamField path="disabled" type="boolean" default="false">
  Disables the password input
</ParamField>

<ParamField path="readOnly" type="boolean" default="false">
  Makes the password input read-only
</ParamField>

<ParamField path="className" type="string">
  Additional CSS classes to apply
</ParamField>

<ParamField path="tagline" type="React.ReactNode">
  Helper text displayed below the label
</ParamField>

## Password Strength Validation

When `isNew={true}`, the component automatically validates and displays these criteria:

1. **Minimum 8 characters** - Password must be at least 8 characters long
2. **One uppercase letter** - Must contain at least one capital letter (A-Z)
3. **One number** - Must contain at least one digit (0-9)
4. **One special character** - Must contain at least one of: `!@#$%^&*)(+=._-`

Each criterion shows a checkmark when met and an X when not met.

## Behavior

- Click the eye icon to toggle password visibility
- Password visibility resets when mouse leaves the input area
- Automatically sets appropriate autocomplete attributes
- Uses `current-password` for existing passwords
- Uses `new-password` when `isNew={true}`

<Note>
  The password strength validation is displayed in French. The validation checks: minimum length, uppercase letters, numbers, and special characters.
</Note>

<Warning>
  Always validate password strength on the server side. Client-side validation improves UX but should not be the only security measure.
</Warning>
