## Overview

The Form package (`@jeremiemeunier/form`) provides a complete set of form components for building interactive user interfaces. Each component follows a consistent state management pattern using the `content/setContent` props, making it easy to handle form data, validation, and error states.

```bash
npm install @jeremiemeunier/form
```

## Basic Usage

All form components follow a consistent pattern:

```jsx
import { Input } from '@jeremiemeunier/form';
import { useState } from 'react';

function MyForm() {
  const [emailContent, setEmailContent] = useState({
    error: false,
    message: '',
    value: ''
  });

  return (
    <Input
      label="Email"
      content={emailContent}
      setContent={setEmailContent}
      type="email"
      required
    />
  );
}
```

## Content State Interface

Most form components use the `InputContent` interface:

```typescript
interface InputContent {
  error: boolean;    // Whether the field has an error
  message: string;   // Error or helper message
  value: any | null; // The field value
}
```

Specific components may extend this interface with additional properties or type constraints.

## Common Props

Many form components share these common props:

<ParamField path="label" type="string">
  Label text displayed above the input
