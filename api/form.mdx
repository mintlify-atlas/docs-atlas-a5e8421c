## Overview

This package provides a complete set of form components with built-in state management, validation, and error handling. All form components use a controlled state pattern with `content` and `setContent` props.

## Components

### Form

Main form container component.

```typescript
interface FormProps {
  children: React.ReactNode;
  handler: (event: FormEvent<HTMLFormElement>) => void;
  encType?: FormEncType;
  className?: string;
}
```

### Input

Standard text input component with validation.

```typescript
interface InputContent {
  error: boolean;
  message: string;
  value: any | null;
}

interface InputProps {
  content: InputContent;
  setContent: React.Dispatch<React.SetStateAction<InputContent>>;
  label?: string;
  size?: number;
  readOnly?: boolean;
  tagline?: React.ReactNode;
  type?: HTMLInputTypeAttribute;
  maxLength?: number;
  placeHolder?: string;
  disabled?: boolean;
  regex?: RegExp;
  regexLabel?: string;
  required?: boolean;
  name?: string;
  autofocus?: boolean;
  autoComplete?: HTMLInputAutoCompleteAttribute;
  className?: string;
  min?: number;
  max?: number;
  step?: number;
  dataIsLoading?: boolean;
}
```

**Usage:**

```tsx
import { Input } from '@jeremiemeunier/form';

const [emailContent, setEmailContent] = useState({
  error: false,
  message: '',
  value: ''
});

<Input
  content={emailContent}
  setContent={setEmailContent}
  label="Email"
  type="email"
  required
/>
```

## State Management Pattern

Most form components follow this state pattern:

```tsx
const [content, setContent] = useState<InputContent>({
  error: false,
  message: '',
  value: null
});

<Component
  content={content}
  setContent={setContent}
  // ... other props
/>
```
